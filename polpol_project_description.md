# polpol - Current Project Description (2026.03.17)

**polpol**은 인천 폴리텍 II 대학 AI융합소프트웨어과 학생들의 성공적인 취업과 원활한 학과 생활을 돕기 위해 설계된 올인원 커뮤니티 플랫폼입니다. 초기 기획안을 바탕으로 실제 성능과 확장성을 고려하여 구체화된 아키텍처로 구현되었습니다.

## 1. 주요 핵심 기능 (Implementation Progress)

### 🚀 취업 및 경력 관리
- **이력서 및 포트폴리오 시스템:** 개별 프로젝트, 경력, 기술 스택을 체계적으로 관리하고 PDF 등 파일 업로드 지원.
- **지원 트래커:** 실시간으로 지원 상태(서류합격, 면접 등)를 추적하고 메인 페이지에서 학우들과 투명하게 공유 (HackerNews 스타일).
- **채용 공고 이원화:** 
  - **기업 구인 (Corporate):** 외부 기업의 실제 채용 공고 관리.
  - **프로젝트 구인 (Project):** 학과 내 졸업 작품, 공모전 팀 구성을 위한 실시간 파트너 모집.

### 🏠 캠퍼스 특화 소셜 서비스
- **커뮤니티 (Post):** 전문적인 질문(AI/SW), 일상 잡담, 학생회 공지 등을 카테고리별로 통합 관리.
- **polbook (Trade):** 전공 서적 및 기자재 전용 중고 장터. 교내 주요 거점(1~7기술관 등) 직거래 장소 지정 및 태그 기반 검색 지원.
- **FAQ 및 정렬 시스템:** 학과 생활 및 취업 관련 자주 묻는 질문을 정렬 우선순위(`sortOrder`)에 따라 체계적으로 제공.

### ⚙️ 개발 생산성 및 인프라
- **자동화된 시딩(Seeding) 시스템:** `dev` 프로필 구동 시 자동으로 DB 테이블을 재구성하고, 실제 서비스와 유사한 고품질 더미 데이터를 삽입하여 즉각적인 테스트 가능.
- **환경별 프로필 관리:** 로컬 개발(`dev`)과 배포 환경을 `application.yml`로 엄격히 분리하여 운영의 안정성 확보.

## 2. 최신 기술 스택 (Tech Stack)

- **Backend:** Java 17, Spring Boot 3.5.0, Spring Data JPA, Spring Security (JWT), MySQL.
- **Frontend:** React, TypeScript, Tailwind CSS, Lucide Icons, Axios.
- **DevOps:** Docker / Docker Compose (Local Database), AWS S3/EC2/RDS (배포 인프라 설계 완료).

### 6. 중고 거래 페이지
- 주로 전공/교양/자격증 서적을 지원하고, 물통, 책 받침대, 프린터기 등 다양한 물품 또한 거래할 수 있다.
-  게시글 필드
    - 기본 정보: 책 제목, 판매 가격, 상세 설명, 사진 첨부
    - 상세 정보(선택): 학년, 학기, 강의명, 교수명
    - 상태 정보: 책 제본/상태(S, A, B급 등), 필기 여부
    - 본관, 1기술관, 2기술관, 3기술관, 백두관, 5기술관, 5기술관 2층 휴게실, 6기술관, 7기술관, 하이테크관, 한라관, 학생회관, 산학협력관 등의 입구로 거래 장소를 한정한다.
- 0원부터 거래 가능하다.
- 에스크로를 적극 도입한다.
- 구매자와 판매자가 금액을 신속하게 주고 받는 것을 지향한다.
- 사업자등록증(임대차계약서가 필요한)이 필요한 서비스는 지양한다.
- 신고 기능
	- 대상
		- 노쇼, 타 물품 판매, 사기 의심, 불쾌한 언행
	- 누적 신고 횟수(예: 3회 이상) 도달 시 경고 조치 및 일정 기간 서비스 이용(글쓰기/채팅) 정지
	- 관리자 페이지에서 신고 접수 내역을 관리할 수 있다.
- 매너 온도 / 평점 기능
	- **거래 완료 시 동작:**
	    - 거래가 '거래 완료'로 변경되면 구매자와 판매자 양측에 리뷰/별점 (1~5점) 작성 팝업 또는 알림 전송.
	    - 남겨진 평점은 해당 유저의 프로필(닉네임 옆)에 평균치로 반영되어, 다른 학우들이 거래 상대의 신뢰도를 미리 파악할 수 있음.
- 전체 시나리오
	- 판매자는 로그인하여 교재 등 물품을 등록한다.
	- 구매자는 강의명/교수명/책 이름 등으로 검색하거나 필터링한다.
	- 구매자는 원하는 물품에 찜하기를 하거나 경제 대금을 예치하고, 판매자와의 채팅을 요청한다.
	- 판매자는 구매 요청 알림을 받는다. (판매 게시글의 상태는 '예약'으로 변경된다.)
	- 판매자와 구매자는 채팅을 하고, 오프라인에서 거래를 완료한다.
	- 구매자가 구매 확정하면 예치금이 판매자에게 송금된다.
	- 송금이 완료되면 게시글의 상태가 거래완료로 변경된다.
	- 거래가 완료되면 판매자와 구매자는 알림을 받아 서로의 평점을 부여한다.
	- 예외 상황 발생 시 신고할 수 있다.
	- 관리자는 신고 내역을 확인하고 처리할 수 있다.
- 판매자 시나리오
	- **목표: 다 쓴 전공 서적을 빠르고 안전하게 판매하여 용돈을 마련하고 싶다.**
	
	1. **로그인 및 책 등록 시작**
	    - Polbook 앱/웹에 교내 이메일 계정으로 자동 로그인된 상태로 접속합니다.
	    - 메인 홈 화면 우측 하단의 **[+ 판매 책 등록]** 플로팅 버튼을 클릭합니다.
	2. **상세 정보 입력 및 등록**
	    - 휴대폰 카메라로 책의 앞/뒷면과 필기 상태를 찍어 사진을 첨부합니다.
	    - 책 제목, 카테고리(`전공 서적`), 학년/학기, 강의명, 교수명, 판매 희망 가격을 입력합니다.
	    - 판매자가 주로 생활하는 **특정 교내 핫스팟**(예: `5기술관 2층 휴게실`)을 거래 희망 장소로 선택 후 [등록하기]를 누릅니다.
	3. **구매자 연락 및 안전 거래 약속**
	    - 구매자로부터 "책 구매 가능할까요?"라는 채팅 알림(Push Notification)이 옵니다.
	    - 채팅을 통해 시간과 장소(5기술관 2층 휴게실)를 정하고, 구매자가 플랫폼 내 **[안전결제(에스크로)]**를 통해 책값을 미리 결제(시스템 보관)했다는 알림을 받습니다.
	    - 다른 사람이 더 이상 구매 요청을 할 수 없도록 게시글 상태를 **'예약중'**으로 확인/변경합니다.
	4. **직거래 도서 전달 및 구매 확정 (정산)**
	    - 약속된 시간에 교내 건물 로비에서 만나 책을 건네줍니다.
	    - 구매자가 현장에서 책을 확인한 뒤 **[구매 확정]** 버튼을 누르면, 시스템에 보관되어 있던 결제 대금이 판매자의 시스템 지갑/계좌로 안전하게 정산됩니다.
	    - 앱을 켜서 상태를 **'거래완료'**로 변경하고, 구매자에 대한 매너 온도(별점 5점)를 남깁니다.
- 구매자 시나리오
	- **목표: 새 학기에 필요한 비싼 전공 서적을 저렴한 가격에, 교내에서 당일 직거래로 구하고 싶다.**

	1. **도서 검색 및 탐색**
	    - 앱을 켜고 메인 홈 화면 검색창에 이번 학기 수강할 `강의명` 혹은 `교수명`이나 `책 제목`을 검색합니다.
	    - 필터 기능을 통해 `전공 서적`, 학과, 학년 등을 조건으로 걸어 목록을 좁힙니다.
	    - 원하는 책을 발견하면 상세 페이지에 들어가 필기 여부와 사진, 판매자가 정해둔 거래 장소(`제1공학관 로비`)를 확인합니다.
	2. **찜하기 혹은 안전결제 구매 요청**
	    - 당장 사지 않더라도 나중에 보기 위해 하트(♡) 버튼을 눌러 **'찜 목록'**에 담아둡니다.
	    - 확정이 섰을 때 **[안전결제(에스크로)]** 버튼을 눌러, 내 계좌/카드 등을 통해 Polbook 시스템에 책값을 안전하게 예치(보관)해 둡니다.
	    - 하단의 **[채팅하기]** 버튼을 눌러 판매자에게 연락합니다.
	3. **오프라인 거래 및 구매 확정(송금 승인)**
	    - 채팅으로 오후 3시에 5기술관 2층 휴게실에서 만나기로 합의합니다.
	    - 현장에서 책을 받아보고, 사진과 동일하고 훼손이 없는지 꼼꼼하게 상태를 확인합니다.
	    - 이상이 없음을 확인한 뒤 그 자리에서 앱 내의 **[구매 확정]** 버튼을 누릅니다. 비로소 보관되던 내 결제 대금이 판매자에게 안전하게 송금됩니다.
	    - 판매자가 '거래완료' 처리를 하면 리뷰 작성 알림이 오며 판매자의 매너 온도에 별점을 부여합니다.
- 관리자 시나리오
	- **목표: 안전하고 깨끗한 교내 중고 거래 플랫폼 환경을 유지하고 불량 유저를 제재한다.**

	1. **신고 접수 및 대시보드 확인**
	    - 관리자 전용 웹 페이지(Admin Dashboard)에 로그인합니다.
	    - 접수된 '신고 목록' 메뉴에 들어갑니다. 누군가 채팅방에서 욕설을 하거나, 노쇼(약속을 잡고 안 나타남)를 한 유저에 대한 신고 건이 접수되어 있습니다.
	2. **신고 내용 검토**
	    - 신고된 유저의 채팅 내역(권한이 있는 경우)이나 상대방이 캡처하여 올린 사유를 확인합니다.
	    - 해당 유저가 과거에도 유사한 사유로 누적 신고 3회를 받은 이력을 확인합니다.
	3. **제재 조치(패널티) 부여**
	    - 해당 악성 유저의 상태를 `정상` ➔ `한 달 이용 정지`로 변경 버튼을 눌러 처리합니다.
	    - 플랫폼 내에서 다른 선량한 학생들이 피해를 보지 않도록 즉각 조치하며, 신고자에게는 "신고하신 사항이 정상 처리되었습니다"라는 알림을 전송하여 신뢰도를 높입니다.
